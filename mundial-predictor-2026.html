Sistema predictivo reproducible para el Mundial 2026

Predice resultados, actualiza probabilidades con marcadores reales, simula el torneo completo y genera narrativas explicables con IA bajo control de costos.

Tesis del proyecto: construir una plataforma de predicción deportiva que combine un núcleo estadístico reproducible con una capa de explicación narrativa, sin depender de llamadas LLM por usuario y sin presentar el sistema como una herramienta de apuestas.

Qué Es Y Qué No Es

El proyecto se entiende mejor si se separan sus capas. El predictor base es estadístico y reproducible; la IA ayuda a explicar contexto, no reemplaza la predicción validada.

Núcleo ML

Predicción reproducible

Calcula probabilidades 1X2 con un ensemble de ELO, Poisson y XGBoost calibrado. Puede correr sin API keys, sin agentes y sin conexión a servicios LLM.

Simulación

Torneo completo

Usa Monte Carlo para estimar avance por rondas, campeón probable y escenarios de grupos a medida que entran resultados reales.

IA explicable

Narrativas controladas

Genera narraciones precomputadas y respuestas de chat con contexto del torneo. La capa IA tiene caché, rate limit, RAG y fallback cuando no hay presupuesto.

No es una herramienta de apuestas. El sistema no recomienda apostar ni promete precisión futura. Sus salidas son probabilidades, simulaciones y explicaciones para análisis de producto, datos y experiencia de usuario.

Arquitectura Del Sistema

El diseño evita mezclar responsabilidades: el modelo predice, el simulador proyecta, la IA explica y los agentes experimentales enriquecen algunos partidos cuando hay presupuesto.

Pipeline predictivo

El pipeline carga resultados históricos, normaliza nombres de selecciones, calcula ELO cronológico, construye features, entrena/calibra modelos y exporta JSON estáticos para el frontend.

results.csv
  -> ELO + features
  -> XGBoost calibrado + Poisson + ELO
  -> Ensemble 22/58/20
  -> live_predictions.json

Razón del ensemble

XGBoost captura patrones no lineales, Poisson aporta distribución de goles y ELO entrega una señal estable de fuerza relativa. La mezcla reduce dependencia de un único modelo.

Pesos actuales documentados: 22% ELO, 58% Poisson, 20% XGBoost.

Narraciones precomputadas

Las narrativas se generan una vez por jornada y quedan en JSON. En fase de grupos se prioriza el tono base de Bogotá para mantener estabilidad antes de activar variaciones regionales.

Chat con contexto del torneo

El chat inyecta partidos del día, tablas y RAG cuando hay embeddings disponibles. Tiene filtro temático, caché y límite por IP para bajar costo y reducir respuestas fuera de dominio.

Capa separada y experimental

El debate de agentes analiza presión de grupo, momentum, localía, dificultad del rival y escenarios de clasificación. No debe confundirse con el modelo ML base.

Uso responsable

Los agentes pueden aportar contexto y marcador exacto candidato, pero su impacto histórico todavía debe medirse. Si falla la API o el presupuesto, el sistema vuelve al ensemble determinístico.

Qué se corrigió en la narrativa

  • Se eliminó el tono de promesa absoluta y se reemplazó por evidencia medible.
  • Se separó modelo base, narrador IA y Agent Debate.
  • Se evita presentar accuracy como única medida de calidad.
  • Se declara la incertidumbre de datos vivos y marcadores recientes.
  • Se reemplaza la orientación de apuestas por análisis explicable.

FinOps IA Y Aprendizaje De Agentes

La IA no solo debe explicar bien: también debe mostrar cuánto cuesta y si mejora con retroalimentación real. Este bloque usa datos registrados en logs y resultados jugados hasta la última revisión local.

Control de costos

Por qué no se abusa del LLM: cache, RAG y rate limit antes de cada llamada

El Chat IA y el Narrator AI no llaman a DeepSeek por cada pregunta de cada usuario. Tres filtros bloquean el costo antes de llegar al modelo, y un cuarto guardia corta el gasto agregado si se acerca al límite diario o mensual.

FinOps actual + proyección

USD por fase operativa

Gasto operativo a hoy: USD 0.51. Costo completo estimado por partido: USD 0.11-0.14. Si se corre el stack completo para los 76 partidos restantes, el total proyectado hasta la final queda cerca de USD 8.87-11.15.

El Agent Debate domina el costo porque usa deepseek-reasoner, hace 9 llamadas por partido y el pensamiento interno del modelo también consume tokens facturables aunque no se muestre al usuario. Narrator AI y los agentes de enriquecimiento usan deepseek-chat-v3-flash/deepseek-chat, mucho más económicos.

Costo restante del torneo

USD por modelo DeepSeek y fase

44 partidos de grupos + 32 de eliminatorias restantes, divididos por el modelo que realmente consume el gasto.

Cómo se genera cada predicción

Agent Debate: 3 agentes, 3 rondas, 1 consenso, 4 predicciones

Feedback real

4 predicciones por debate

Cada partido con Agent Debate produce 4 lecturas: Group Analyst, Tactical Scout, Sentiment Reader y Consensus. La medición actual evalúa 4 partidos ya jugados.

No es fine-tuning automático: es aprendizaje operativo. Los resultados reales alimentan evaluación, ajuste de prompts, pesos y reglas por jornada.

Operación Durante El Mundial

La operación diaria debe mantener tres artefactos alineados: resultados reales, predicciones recalculadas y narrativas actualizadas.

Ciclo diario recomendado

python scripts/live_update.py
python scripts/predict_live.py --export
python scripts/precompute_narrations.py
cd frontend
npx vercel --prod

J2 y J3 requieren más cuidado

En J2, los partidos del segundo bloque deben leer los resultados del primer bloque. En J3, los partidos de cada grupo se juegan al mismo tiempo y la narrativa debe enfocarse en clasificación directa, diferencia de gol y mejores terceros.

Limitaciones Serias

Declarar límites hace que el proyecto se vea más maduro, no menos ambicioso.

Varianza deportiva

Un partido único puede girar por expulsiones, lesiones, clima o decisiones arbitrales. Las probabilidades no son certezas.

Datos vivos

Si la fuente de resultados llega tarde o el fixture cambia, la app puede mostrar contexto desactualizado hasta el siguiente ciclo.

LLM y narrativa

Las respuestas generativas pueden equivocarse si el input está incompleto. Por eso se prefiere contexto explícito, cache y fallback.
