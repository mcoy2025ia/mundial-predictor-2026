Laboratorio multiagente para razonar el Mundial 2026

Varios agentes predicen el mismo partido desde lógicas distintas, compiten contra un modelo estadístico base y son evaluados contra el resultado real para ajustar confianza, pesos y uso de modelos por fase.

Tesis del proyecto: demostrar que una arquitectura de agentes puede entender la lógica cambiante de un Mundial, usar modelos DeepSeek según complejidad, controlar costos con cache y precomputación, y convertir cada resultado real en feedback medible.

Qué Demuestra El Proyecto

El fútbol es el entorno de prueba; el producto demostrado es una arquitectura de agentes evaluables, con costos controlados y lógica distinta por fase del torneo.

Benchmark estadístico

Modelo base medible

El ensemble ELO + Poisson + XGBoost produce una predicción reproducible y barata. No compite por protagonismo: sirve como punto de comparación contra los agentes.

Arquitectura multiagente

Varios razonamientos, mismo partido

Cada agente mira una señal distinta: presión de grupo, táctica, ánimo o consenso. Todos dejan una predicción estructurada que luego se evalúa contra el marcador real.

FinOps IA

Gastar donde aporta

DeepSeek Reasoner se reserva para debates complejos. Narraciones, predicciones y contexto quedan cacheados o precomputados para evitar llamadas LLM por cada clic.

No es una app para apostar. Es una demostración de buenas prácticas: orquestación de agentes, uso selectivo de modelos, trazabilidad de predicciones, evaluación posterior y control de consumo.

Arquitectura De Agentes Evaluables

El diseño separa responsabilidades: el modelo base entrega el benchmark, los agentes generan hipótesis, el evaluador compara contra resultados reales y FinOps decide cuándo usar modelos caros.

Benchmark reproducible

El pipeline estadístico calcula una predicción base sin depender de LLM. Esa salida permite saber si los agentes realmente agregan valor o solo generan una explicación más bonita.

results.csv
  -> ELO + features
  -> XGBoost calibrado + Poisson + ELO
  -> Ensemble 22/58/20
  -> benchmark para agentes

Por qué sigue siendo necesario

El modelo base es barato, estable y auditable. Si los agentes fallan, no hay presupuesto o no existe contexto suficiente, el sistema mantiene una predicción mínima confiable.

El benchmark evita que el caso se vuelva puro storytelling: cada agente debe ganarse su peso con resultados.

Narraciones precomputadas

Las narrativas no se generan cada vez que alguien abre la app. Se calculan por jornada, quedan en JSON y se actualizan cuando cambian resultados, presión de grupo o contexto competitivo.

RAG cerrado y caché

El chat no consulta un RAG abierto sin control. Usa contexto del torneo, filtro temático, caché y límites por IP para evitar alucinaciones, respuestas fuera de dominio y consumo repetido.

Cuatro hipótesis trazables

Group Analyst, Tactical Scout, Sentiment Reader y Consensus predicen el mismo partido. Cada salida queda guardada para evaluar 1X2, marcador exacto y desempeño por jornada.

Aprendizaje operativo

Los agentes no aprenden mágicamente: el sistema compara predicción vs resultado, identifica quién acertó más y ajusta pesos, prompts y modelos según la fase del Mundial.

Evaluación multiagente

  • Cada partido debatido guarda 4 predicciones: 3 agentes + consenso.
  • Después del resultado, se evalúa 1X2, marcador exacto y desempeño por jornada.
  • La muestra actual es temprana: 4 partidos jugados con debate evaluable.
  • Group Analyst lidera la muestra inicial en J2 porque entiende puntos, presión y clasificación.
  • La decisión futura no es “usar más IA”, sino usar el agente correcto en la fase correcta.

El bloque caro no es la narración: es el Agent Debate. Reasoner domina porque cada partido debatido dispara 9 llamadas y cobra tokens de pensamiento interno. El resto se contiene con deepseek-chat-v3-flash, deepseek-chat, cache y precomputación.

Control de costos

Por qué no se abusa del LLM: cache, RAG y rate limit antes de cada llamada

El Chat IA y el Narrator AI no llaman a DeepSeek por cada pregunta de cada usuario. Tres filtros bloquean el costo antes de llegar al modelo, y un cuarto guardia corta el gasto agregado si se acerca al límite diario o mensual.

FinOps actual + proyección

USD por fase operativa

Gasto operativo a hoy: USD 0.51. Costo completo estimado por partido: USD 0.11-0.14. Si se corre el stack completo para los 76 partidos restantes, el total proyectado hasta la final queda cerca de USD 8.87-11.15.

El Agent Debate domina el costo porque usa deepseek-reasoner, hace 9 llamadas por partido y el pensamiento interno del modelo también consume tokens facturables aunque no se muestre al usuario. Narrator AI y los agentes de enriquecimiento usan deepseek-chat-v3-flash/deepseek-chat, mucho más económicos.

Por qué domina Reasoner

1 partido debatido = 9 llamadas caras

Cada agente no solo responde una vez: razona, rebate y participa en el consenso. Esa arquitectura da trazabilidad, pero concentra el costo en deepseek-reasoner.

La operación diaria mantiene alineados cuatro artefactos: resultado real, benchmark ML, pronósticos de agentes y narrativas cacheadas.

J2 y J3 requieren más capacidad

En J2, los agentes deben leer presión real: quién necesita ganar, quién puede empatar y quién se complica. En J3, Reasoner se justifica porque aparecen simultaneidad, diferencia de gol y pelea por mejores terceros.

Limitaciones Serias

La arquitectura es potente precisamente porque declara cuándo no sabe, cuándo no debe gastar y cuándo una predicción todavía no tiene muestra suficiente.

Muestra pequeña

El desempeño inicial de agentes no debe venderse como verdad definitiva. Cada jornada aumenta la evidencia y reduce ruido.

Datos vivos

Si resultados, horarios o tablas llegan tarde, los agentes pueden razonar sobre contexto viejo hasta el siguiente ciclo de actualización.

LLM no es oráculo

Reasoner ayuda en escenarios complejos, pero no reemplaza evaluación. Toda salida debe quedar guardada y compararse contra el resultado real.
